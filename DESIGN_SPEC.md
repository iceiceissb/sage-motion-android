# 方案 3：Calm Veil 设计与动效规范

## 设计目标

让 AI 的过程像一层安静、可读的薄雾叠在真实环境之上：不抢夺用户注意力，但能回答“AI 现在在做什么、为什么这么判断、哪里不确定、我能否接管”。

## 固定布局

- 竖屏手机，主场景为全屏地图或相机画面。
- 顶部：半透明白色 AI 状态卡；实验时长按 1 秒进入研究员控制台。
- 中部：场景内容和语义动效，不弹出阻断式加载页。
- 底部：固定高度的白色结果面板，主按钮位置跨条件一致。
- 主色 `#3F6F5C`，正文 `#26332F`，不确定性使用低饱和赭色 `#B86B2C`。
- 圆角 17–30dp；状态卡 22dp；按钮高度 52–54dp。

## 条件增量

| 能力 | Baseline | Semantic Motion | SAGE Full |
|---|---|---|---|
| 通用等待反馈 | 有 | 有 | 有 |
| AI 阶段名称 | 无 | 有 | 有 |
| 通用载体显影（不解释 AI 状态） | 有 | 有 | 有 |
| 与 AI 阶段一一对应的语义动效 | 无 | 有 | 有 |
| 不确定性与信息缺口 | 无 | 无 | 有 |
| 查看判断依据 | 无 | 无 | 有 |
| 换路线/重试等接管 | 最小 | 最小 | 完整 |

禁止为某一条件单独增加更漂亮的页面、更多结果内容或更短等待时间；否则会混入视觉质量和信息量差异。

## 动效规则

- 唤起：800ms，轻量进度指示。
- 输入感知：1200ms。定位显示双路线从同一起点分叉；视觉显示镜头接入与扫描；语音显示呼吸式语音球。
- 推理：1600ms。路线显示两候选独立计算头与依据汇入，识别显示候选区域和特征比较，语音显示由内向外的理解/回答方向。
- 不确定性：1000ms，仅 SAGE Full 明确显示信息缺口；其他条件维持等时等待，不泄露语义。
- 完成：550ms，底部结果面板淡入并上移。
- 使用 `ease` 感受的缓和节奏，不使用弹跳、抖动或高频闪烁。

## 语义运动词汇（v0.8）

- `Trail`：路线逐段显影、方向性虚线和移动光点，表达定位、推进和规划中的空间连续性。
- `Seed`：少量特征点或依据节点由散到聚，表达识别与比较；节点数量保持克制，避免装饰性粒子噪声。
- `Veil`：扫描幕、透明度和虚实层级表达感知范围；不确定时不强行“落定”，使用未闭合、虚线和低频呼吸。
- `Listening`：同心低对比波纹与非匀速声波条；不用持续缩放布局，避免界面抖动。
- `Responding`：由中心向外/向上扩散的短线层，表达组织完成后的输出，而不是继续推理。
- `Gather`：路线、照片和问答从原位置沿可追踪轨迹汇聚，表达素材被回收而非凭空生成。
- `Weave`：素材先归组、再接入时间轴、最后形成卡片；分阶段运动表达“汇总 → 生成 → 编排”。
- `Afterglow`：旧路线不瞬间消失，而以低对比残影保留，帮助用户理解新旧方案的因果关系。
- `Fork / Morph`：变化点先被标记，旧路径再分叉并连续弯折为新路径，表达动态重规划而非页面替换。
- `Common Fate`：同属一条路线或同一段回忆的元素以同方向、同节奏移动，利用共同命运形成感知分组。
- `Competition / Convergence`：两条候选路线拥有独立显影头、进度和证据流；完成时推荐路线加粗，备选路线仍保留，表达比较而非“凭空给答案”。
- `Live Coupling`：实际语音时，麦克风 RMS 音量直接控制语音球和波形幅度；运动响应真实输入而非播放固定循环。
- `Material Satellites`：照片、语音与重规划次数决定游记汇聚时的素材卫星数量，使生成动效与本次旅程数据一致。
- `Semantic Bubbles`：多点识别线索以低频漂浮气泡附着在照片的不同位置；气泡标明“语义候选”和置信度，不伪装成模型并未提供的精确检测框。
- `Lasso Focus`：用户拖动形成连续圈选轨迹，圈线完成后以低频光晕确认关注范围，随后推荐问题从圈选动作连续进入提问面板。
- `Constraint Funnel`：路线生成前先收集点选、文字或语音约束，约束在计算阶段压缩成顶部摘要，使“人的意图 → AI 双路线比较”因果可见。
- `Route Story`：知识游记以实际路线作为叙事主线，照片按发生顺序落到沿线节点；节点问题数量作为可见徽标，点击后详情面板连续切换到对应问答，维持空间位置与知识内容的关联。
- 场景之间使用短距离平移与淡入淡出保持空间连续性；结果面板固定从底部进入。
- Baseline 只显示非语义通用反馈；Semantic/SAGE 才显示可辨识状态运动，SAGE 额外显示不确定性与接管线索。

## 先规划、后并行的任务映射

信息架构不是线性向导。A 是探索前的唯一必经步骤；采纳路线后进入常驻探索工作台，B1、B2、D 是可重复的并行工具。每次工具完成都返回工作台，过程照片、转写与重规划次数持续累积；C 只由用户点击“结束探索”触发。

### A 环境感知

`人提供点选/文字/语音约束 → 唤起 → 定位 → 比较遮阴/座椅 → 标记信息缺口 → 推荐路线`

SAGE Full 显示实线路线、虚线备选路线、遮阴数据缺口和“查看依据/换路线”。
所有条件都保留双路线逐段显影，避免 Baseline 一次成型；Semantic/SAGE Full 进一步显示计算头、证据节点、竞争进度和信息缺口。

### B1 视觉探索

`唤起 → 取景识别 → 多点语义气泡 → 用户圈选 → 推荐/自定义提问 → 圈搜解答 → 返回工作台`

使用用户实际拍摄照片，端侧模型识别人物、动物、建筑、食物、自然等通用类别，不再预设为花卉。扫描框是动态界面状态，不替代真实图像素材；悬浮气泡是语义线索的空间化呈现，当前模型不返回目标坐标，因此不能作为精确定位证据。

### B2 语音陪伴

`唤起 → 应用内聆听与转写 → 理解问题 → 组织回答 → 播报 → 返回工作台`

离线实验模式保留固定转写文本和回答；生态演示可调用 Android 系统语音识别，回答由设备 TextToSpeech 播报。两种模式在日志中区分。

### C 创作与分享

`唤起 → 路线逐段显影 → 照片落入沿线节点 → 绑定节点问答 → 编排与检查 → 点击回看/保存/分享`

使用 Gather / Weave / Route Story：素材从各自来源连续汇聚到空间路线，照片节点保留问题数量；编辑完成后，用户点击节点即可回看这张照片下的全部问题和回答。

### D 动态调整

`唤起 → 检测变化点 → 保留旧路线残影 → 生成绕行分支 → 权衡并确认 → 返回工作台`

使用 Afterglow / Fork / Morph：旧路线不突变消失，封闭点产生警示波纹，新路线围绕变化点连续弯折，完成时仍保留用户接管备选。

## 动效研究约束

- 采用 staged transition：一次只解释一个主要变化，降低对象追踪负担。
- 相关对象使用共同方向和节奏，避免无意义粒子随机运动。
- 永久性状态改变使用一次性显影；只有“正在感知/行进”等持续过程使用循环运动。
- 支持系统动画缩放设置；Baseline 保留等时但不显示语义运动。

## 无障碍与实验控制

- 状态不只依赖颜色，同时提供文字和形态变化。
- 主要正文不小于 14sp；辅助说明最低 11sp。
- 核心按钮触控高度不低于 48dp。
- 参与者页面不显示 C0/C1/C2 名称。
- 错误、取消和重新开始都必须回到可恢复状态。
