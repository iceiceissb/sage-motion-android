# SAGE 实验 Demo 缺口与联网 Agent 说明

## 当前可用基线（v0.7.0）

- 三种动效条件与 A/B1/B2/C/D 任务可完整演示；A 后进入并行探索工作台，B1/B2/D 可重复调用，C 作为结束生成动作。
- 离线实验模式使用固定刺激、固定结果和固定状态时序。
- 会话可恢复，事件实时写入本地 CSV，可查看、单次/批量导出，并可删除单条或全部历史及关联照片。
- 真实相机照片由端侧通用图像标签器分析，过程照片持续累积并进入最终知识游记；语音在应用内直接录音、转写和播报。
- 路线、视觉、语音、游记和重规划均采用状态驱动的语义动效；双路线逐段计算、真实音量耦合与素材数量耦合可作为新一轮预测试刺激。
- 联网 Agent 使用清华校园固定演示点的天气与空气质量，不申请参与者定位。
- 联网请求超时后依次使用 30 分钟新鲜缓存、6 小时备用缓存、离线脚本。
- 在线、缓存、离线回退和视觉固定刺激都在结果界面与日志中明确标记。

## 正式实验前的 P0 缺口

1. 冻结唯一研究问题和自变量。不要在同一轮同时改变动效语义、不确定性表达、证据入口和拟人程度。
2. 为 A/B/C 条件制作内容完全匹配的任务材料，并加入高置信、低置信和失败边界三类 ground truth。
3. 在 App 内加入逐任务量表、状态识别题、决策时间与误操作记录；目前只有交互事件日志。
4. 除条件 Latin square 外，还需要任务内容组的随机化/平衡，避免任务顺序和学习效应混入条件效应。
5. 补齐正式同意、退出与隐私说明；“删除本次/全部本地数据”已经实现，仍需在预测试中验证参与者是否理解删除范围。
6. 固化研究员口播、设备型号/刷新率/亮度、场地噪声与光照记录。

## 正式实验前的 P1 缺口

- 5–8 人预测试，检查户外可见性、动效含义识别、眩晕/疲劳和任务时长。
- Release 签名、版本冻结、APK SHA-256、实验配置清单与变更记录。
- 已启用真实相机/麦克风，需要继续补齐永久拒绝后的系统设置引导、电话中断测试与更完整的数据最小化说明。
- 导出包增加配置清单、问卷数据和校验和，形成可复现实验数据包。

## 模式隔离原则

- `实验模式 · 离线固定`：用于正式被试实验和 A/B/C 统计比较。
- `联网 Agent · 生态演示`：用于展示产品可能性，不并入正式实验统计。
- 离线正式实验仍可使用固定视觉刺激；生态演示可拍摄真实照片并以“实际拍照 · 端侧 ML Kit”标记，两类数据不得混入同一统计条件。

## API 架构

`ExperimentViewModel -> AiDemoApi -> MockAiDemoApi / ParkAgentApi -> OpenMeteoParkContextProvider`

首个联网适配器只接无需密钥的 Open-Meteo。需要密钥的大模型与路线服务必须经过服务端代理；禁止把 Gemini、OpenAI 或 openrouteservice 密钥写入 APK、资源文件或 GitHub。
